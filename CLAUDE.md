# CLAUDE.md

## Project Overview

FlowPilot is a production-ready SaaS landing page for an AI-powered workflow automation platform. Built with Next.js 16, React 19, and Tailwind CSS 4. Features include dark/light theme, animated UI, waitlist form with API validation, FAQ accordion, pricing toggle, and full SEO/accessibility compliance.

## Tech Stack

- **Framework:** Next.js 16 (App Router)
- **UI:** React 19, Tailwind CSS 4, Framer Motion
- **Language:** TypeScript 5.8 (strict mode)
- **Backend:** Next.js API Routes, Prisma 7.8, PostgreSQL
- **Auth:** NextAuth v5 (beta)
- **Validation:** Zod 4.4, React Hook Form
- **Testing:** Jest 30, React Testing Library
- **Linting:** ESLint 9, Prettier 3.9

## Coding Conventions

- Use TypeScript strict mode — no `any` types
- Functional components only (no class components)
- Use `'use client'` only when component requires browser APIs, hooks, or event handlers
- Use `@/*` path alias for imports from `src/`
- Prefer `interface` for object shapes, `type` for unions/intersections
- Use `cn()` utility from `@/lib/utils` for conditional classes
- Tailwind CSS 4 utility classes only — no custom CSS unless necessary
- Follow existing color token system (oklch-based shadcn theme in `globals.css`)

## Commit Conventions

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
<type>: <description>
```

Types: `feat`, `fix`, `docs`, `test`, `refactor`, `chore`, `style`, `perf`

Examples:
- `feat: add team management dashboard`
- `fix: resolve waitlist duplicate detection race condition`
- `test: add Pricing component tests`

## Formatting Rules

Run before committing:

```bash
npx prettier --write .     # Format (auto-fix)
npx eslint .               # Lint
npx tsc --noEmit           # Type check
```

- Prettier with `prettier-plugin-tailwindcss` for class sorting
- No semicolons, single quotes

## Testing

- Test files adjacent to source: `Component.test.tsx` next to `Component.tsx`
- Use `@testing-library/react` for component tests
- Use `@testing-library/user-event` for user interactions
- Run: `npm test` (must pass before committing)
- Existing suites: ThemeToggle, Navbar, Footer, Waitlist, Pricing, FAQ, AnimatedSection, API Route

## Architecture Rules

- **Preserve existing architecture** — do not restructure folders or rename files without explicit approval
- **Prefer reusable components** — extract shared UI into `src/components/ui/` when possible
- **Never modify** files under `src/generated/prisma/` — these are auto-generated by Prisma
- **Ask before** making major architectural changes (new directories, new dependencies, restructuring)
- Use `next/dynamic` for code splitting below-the-fold sections
- Lazy-load heavy components (mobile menu, etc.)

## Project Structure

```
src/
├── app/           # Next.js App Router pages and API routes
├── components/    # React components (layout/, sections/, ui/, providers/)
├── hooks/         # Custom React hooks
├── lib/           # Utilities, constants, validations, Prisma client
├── data/          # Static data (FAQ items)
└── types/         # TypeScript type definitions
```
